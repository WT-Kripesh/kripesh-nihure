# Kripesh Nihure — Portfolio

Personal portfolio and field journal built with [TanStack Start](https://tanstack.com/start), React, and Tailwind CSS v4.

## About Me

AI Engineer building production LLM, RAG and Voice AI systems from the Himalayas. I build enterprise-grade conversational AI — multi-tenant RAG platforms, real-time Voice-to-Voice pipelines, NLP-to-SQL — so I can work from places that remind me how small the world really is.

## Stack

- **Framework:** TanStack Start (React, SSR)
- **Bundler:** Vite
- **Styling:** Tailwind CSS v4 + shadcn/ui
- **Routing:** TanStack Router (file-based)
- **Query:** TanStack Query
- **Language:** TypeScript

## Development

```sh
npm install
npm run dev        # local dev server
npm run build      # production build (outputs to .output/)
npm run preview    # preview production build
```

## Deployment

Deployed to **GitHub Pages** via a GitHub Actions workflow (`.github/workflows/deploy.yml`):

| Trigger | Branch |
|---|---|
| Push | `main` |
| Manual | `workflow_dispatch` |

**Custom domain:** [`kripeshnihure.com.np`](https://kripeshnihure.com.np) (configured via `CNAME`)

### Pipeline

1. Checkout → Node 20 (npm cache)
2. `npm ci` → `npm run build`
3. `actions/configure-pages@v5` sets up Pages
4. Artifact uploaded from `./.output/public`
5. Deployed via `actions/deploy-pages@v4`

The build output directory `./.output/public` is what GitHub Pages serves — it's generated by TanStack Start's Nitro-based static/prerendered build.
